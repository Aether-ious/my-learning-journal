Ring 1 — Borrower-level Aggregation (SAS)

Warm-up refresher exercise

We start with a simple banking-style dataset and aggregate it at borrower + month level. Two SAS approaches are demonstrated: PROC SQL and DATA step.

Sample Data

Borrower_IDLoan_IDBalancePaymentMonth

1001L22000100202401
1002L38000300202401
1002L46000250202401
1003L54000150202401

Task

  • Total_Balance = sum(Balance)
  • Total_Payment = sum(Payment)
  • Num_Loans = count(Loan_ID)
  • Avg_Payment_Ratio = Total_Payment / Total_Balance

Solution — PROC SQL

Use sum(Payment)/sum(Balance) directly. Avoid CALCULATED inside aggregates.
proc sql;
  create table borrower_summary as
  select 
    Borrower_ID,
    Month,
    sum(Balance) as Tot_Balance,
    sum(Payment) as Tot_Payment,
    count(Loan_ID) as Num_Loans,
    sum(Payment) / sum(Balance) as Avg_Pymt_Ratio
  from table1
  group by Borrower_ID, Month;
quit;

Solution — DATA Step + BY-group Processing

proc sort data=table1; by Borrower_ID Month; run;

data borrower_summary;
  retain Tot_Balance Tot_Payment Num_Loans;
  set table1;
  by Borrower_ID Month;

  if first.Month then do;
    Tot_Balance = 0;
    Tot_Payment = 0;
    Num_Loans = 0;
  end;

  Tot_Balance + Balance;
  Tot_Payment + Payment;
  Num_Loans + 1;

  if last.Month then do;
    Avg_Pymt_Ratio = Tot_Payment / Tot_Balance;
    output;
  end;
run;

Alternative — Mean of Loan-Level Ratios

If instead you want the mean of per-loan Payment/Balance ratios:

proc sql;
  create table loan_ratios as
  select Borrower_ID, Month, Payment/Balance as loan_ratio
  from table1;

  create table borrower_avg_ratio as
  select Borrower_ID, Month, mean(loan_ratio) as Avg_LoanLevel_Ratio
  from loan_ratios
  group by Borrower_ID, Month;
quit;

Takeaways

  • Ratio of sums = borrower-level payment capacity.
  • Mean of ratios = average loan-level behavior.
  • Both are valid; use depending on the business question.
  • Avoid typos (balacen → Balance).
